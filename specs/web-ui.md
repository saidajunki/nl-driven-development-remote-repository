# Web UI 仕様（閲覧・可視化）

## 目的

- Git リポジトリの情報を Web で扱えるようにしつつ、特に `specs/` と `rules/` を読みやすくする。

## 主要画面（MVP）

- ランディングページ（未ログイン時）: サービス紹介、アニメーション多め
- ダッシュボード（ログイン後）: リポジトリ一覧
- リポジトリ詳細
  - Specs: `specs/` の目次表示、Markdown 表示、検索
  - Rules: `rules/` の一覧、Markdown 表示、カテゴリ別集計（円グラフ等）
  - Coverage: カバレッジ概要（全体・ブランチ/最新コミット）

## 表示要件（specs）

- Markdown のレンダリング
- 見出しから自動目次
- 変更履歴（最低限: 最新コミットとの差分リンク）※詳細は将来拡張

## 表示要件（rules）

- Markdown のレンダリング
- カテゴリ（security/coding/testing...）の一覧
- カテゴリ割合の可視化
