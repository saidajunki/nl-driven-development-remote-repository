# 概要（目的・スコープ）

## 目的

- GitHub のように「リモートリポジトリ」を提供しつつ、**日本語で書かれがちな仕様書（要件定義書）やルールを中心に管理・閲覧しやすくする**。
- 将来的な AI 駆動開発を見据えつつ、当面は **AI をシステムに組み込まず**、仕様・ルール・品質指標の可視化をまず成立させる。

## ゴール（MVP）

- 自前ホスティングの Git リモートを提供できる（`clone/fetch/push`）。
- リポジトリ内の `specs/`（要件定義書など）と `rules/`（ルール群）を Web で読みやすく表示できる。
- `rules/` の内容を集計し、**ルール種類（例: security/coding/testing ...）の割合**を可視化できる。
- テストカバレッジを（レポート取り込み方式で）表示できる。

## 非ゴール（当面やらない）

- AI（生成/レビュー/質問応答）の組み込み
- SSH での Git 操作（当面は HTTPS を前提）
- PR/MR、Issue、Actions 相当のフル機能（必要に応じて段階導入）

## 前提

- リポジトリ内容は基本「公開」扱いにする（Web で閲覧可能、`clone/fetch` も公開）。
- **push は公開のままにせず**、認証されたユーザーのみ許可する（PAT を利用）。
